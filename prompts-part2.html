GLOBAL SPECIFICATION — READ BEFORE ANY TASK
================================================================
App name:    Tohfa
Backend:     Node.js + SQLite
Auth:        JWT access tokens (15-min expiry) + refresh tokens
             Refresh endpoint: POST /api/auth/refresh
             Token storage: sessionStorage (frontend handles)

Admin auth:  Separate JWT role claim: { role: "admin" }
             All /api/admin/* routes require admin JWT.
             Return 403 if token is valid but role !== "admin".

Payments:    Razorpay only
Currency:    All amounts stored in paise (1 rupee = 100 paise).
             Display as ₹ with 2 decimal places.
             EXCEPTION: The admin frontend renders amounts already
             formatted as strings (e.g. "₹12,450") — the backend
             must return amount_paise (integer) AND
             amount_display (string "₹X,XX,XXX") in every route
             that surfaces money.

API base URL: /api (all routes prefixed with /api)
Auth header:  Authorization: Bearer [token]

ERROR RESPONSE FORMAT (same on every endpoint):
{
  "error": true,
  "message": "Human readable message",
  "code": "ERROR_CODE"
}

SUCCESS RESPONSE FORMAT:
{
  "success": true,

TASK 08: POST /api/admin/auth/login
TYPE: API ROUTE
NEEDED BY: Admin login (separate from buyer/seller login)
================================================================

ENDPOINT:
Method + URL: POST /api/admin/auth/login
Auth required: NO (this IS the auth)
Rate limit: 10 requests per minute per IP

REQUEST:
{
  "username": "string",   // or email — accept either
  "password": "string"
}

RESPONSE (success 200):
{
  "success": true,
  "data": {
    "access_token": "string",     // JWT, 15-min expiry, role: "admin"
    "refresh_token": "string",    // JWT, 7-day expiry
    "admin": {
      "id": 1,
      "username": "admin",
      "display_name": "Tohfa Admin",
      "role": "super_admin"
    }
  }
}

RESPONSE (errors):
  400: username or password missing
  401: invalid credentials (always say "Invalid username or password" — do NOT reveal which field is wrong)
  403: account is inactive (is_active = 0)
  429: rate limit exceeded

BUSINESS LOGIC:
1. Accept either username or email in the "username" field.
2. Look up admin_users by username OR email.
3. If not found → 401.
4. Compare password with bcrypt.compare().
5. If mismatch → 401.
6. If is_active = 0 → 403.
7. Generate access token (JWT) with payload:
     { sub: admin.id, role: admin.role, type: "admin_access" }
     Expiry: 15 minutes.
8. Generate refresh token (JWT) with payload:
     { sub: admin.id, type: "admin_refresh" }
     Expiry: 7 days.
9. Update last_login_at on the admin_users row.
10. Write audit log entry:
     event_type: "admin.session.login"
     actor_id: admin.id
     target_type: "dashboard"
     target_label: "Admin Login"
11. Return 200 with tokens and admin object.

FRONTEND MATCH CHECK:
Admin panel is a web panel (not mobile).
Tokens stored in sessionStorage under keys:
  "tohfa_admin_token" and "tohfa_admin_refresh".
Order Detail screen has: window.location.href = '/login'
when token is missing — ensure the login route matches
the frontend redirect path.

{
  "success": true,
  "data": {
    "seller_id": 1,
    "status": "banned",
    "banned_at": "2025-01-15T14:32:07Z"
  }
}

  401: missing or invalid token
  403: not admin role

        event_type: "admin.seller.banned"
        actor_id: [admin id from token]
        actor_name: [admin display_name]
        target_type: "seller"
        target_id: seller_id
        target_label: "Seller: [display_name]"
        before_json: { "status": "active" }
        after_json: { "status": "banned", "ban_reason": "...", "updated_at": "..." }

  and reels immediately."
ban_reason field is shown in the form before confirming.
after_json shape must match what the audit log diff renders:
  { "status": "active" → "banned", "ban_reason": "...", "updated_at": "..." }
